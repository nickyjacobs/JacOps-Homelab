# n8n

🇬🇧 English | 🇳🇱 [Nederlands](01-n8n.nl.md)

n8n is the workflow automation platform for the homelab. It handles scheduled tasks, API integrations and event-driven automation. It runs as a Docker stack inside an LXC container.

## Why n8n

The homelab needs automation that can talk to external APIs (GitHub, Notion, cloud services) without exposing those APIs to the internal network. n8n handles this through a reverse tunnel to a CDN provider. No ports are forwarded on the gateway, no firewall rules need to allow inbound traffic, and the public URL stays reachable even when the WAN IP changes.

The alternative was running cron scripts directly on a Proxmox node. That works for simple tasks but becomes unmanageable when workflows need conditional logic, error handling and external API calls. n8n provides a visual editor and handles retries, webhooks and credential storage out of the box.

## Architecture

```
Internet ─── CDN Tunnel ─── Cloudflared ─┐
                                         │
                LXC Container (CT 150)   │
                ┌────────────────────────┤
                │  Docker Compose        │
                │  ├─ n8n (port 5678)    │
                │  ├─ PostgreSQL         │
                │  └─ Cloudflared ───────┘
                └────────────────────────
                VLAN 40 (Apps)
```

Three containers run inside a single Docker Compose stack:

- **n8n** handles the workflow engine and web UI
- **PostgreSQL** stores workflow definitions, credentials and execution history
- **Cloudflared** maintains an outbound tunnel to the CDN provider, making n8n reachable on a public URL without opening any inbound ports

The LXC container sits on the Apps VLAN. It can reach the internet (for API calls and the tunnel) but is firewalled from the Servers and Management VLANs. The only exception is a monitoring health check from Uptime Kuma.

## Access

The web UI is accessible through the public tunnel URL. There is no direct LAN access to port 5678. This is intentional: all traffic goes through the tunnel, which provides TLS termination and DDoS protection from the CDN provider.

For local administration, SSH into the Proxmox node and use `pct exec 150 -- bash` to reach the container, then `docker exec -it n8n-n8n-1 sh` for the n8n instance.

## Backup

The container is included in the weekly cluster backup job (Sunday 03:00, zstd, four-week retention). This captures the full container filesystem including the Docker volumes with workflow definitions and the PostgreSQL database.

For additional safety, n8n workflow definitions can be exported as JSON from the web UI.
