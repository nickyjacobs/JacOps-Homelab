# n8n

🇬🇧 [English](01-n8n.md) | 🇳🇱 Nederlands

n8n is het workflowautomatiseringsplatform voor het homelab. Het verwerkt geplande taken, API-integraties en event-driven automatisering. Het draait als Docker stack in een LXC-container.

## Waarom n8n

Het homelab heeft automatisering nodig die met externe APIs kan communiceren (GitHub, Notion, clouddiensten) zonder die APIs aan het interne netwerk bloot te stellen. n8n handelt dit af via een reverse tunnel naar een CDN-provider. Geen poorten worden geforward op de gateway, geen firewallregels hoeven inbound verkeer toe te laten, en de publieke URL blijft bereikbaar zelfs wanneer het WAN IP wijzigt.

Het alternatief was cron scripts direct op een Proxmox node draaien. Dat werkt voor eenvoudige taken maar wordt onbeheersbaar wanneer workflows conditionele logica, foutafhandeling en externe API-calls nodig hebben. n8n biedt een visuele editor en verzorgt retries, webhooks en credential-opslag standaard.

## Architectuur

```
Internet ─── CDN Tunnel ─── Cloudflared ─┐
                                         │
                LXC Container (CT 150)   │
                ┌────────────────────────┤
                │  Docker Compose        │
                │  ├─ n8n (port 5678)    │
                │  ├─ PostgreSQL         │
                │  └─ Cloudflared ───────┘
                └────────────────────────
                VLAN 40 (Apps)
```

Drie containers draaien in een enkele Docker Compose stack:

- **n8n** verzorgt de workflow-engine en web UI
- **PostgreSQL** slaat workflowdefinities, credentials en uitvoeringsgeschiedenis op
- **Cloudflared** onderhoudt een uitgaande tunnel naar de CDN-provider, waardoor n8n bereikbaar is op een publieke URL zonder inkomende poorten te openen

De LXC-container staat op het Apps VLAN. Die kan het internet bereiken (voor API-calls en de tunnel) maar is afgeschermd van de Servers en Management VLANs. De enige uitzondering is een monitoring health check van Uptime Kuma.

## Container specs

| Instelling | Waarde |
|------------|--------|
| VMID | 150 |
| Type | LXC (unprivileged) |
| Node | Node 1 |
| CPU | 2 cores |
| RAM | 2048 MB |
| Disk | 40 GB (HDD) |
| VLAN | 40 (Apps) |
| IP | Statisch, toegewezen via containerconfiguratie |
| Boot | `onboot: 1` |

## Toegang

De web UI is bereikbaar via de publieke tunnel-URL. Er is geen directe LAN-toegang tot poort 5678. Dit is een bewuste keuze: al het verkeer gaat via de tunnel, die TLS-terminatie en DDoS-bescherming biedt van de CDN-provider.

Voor lokaal beheer: SSH naar de Proxmox node en gebruik `pct exec 150 -- bash` om de container te bereiken, gevolgd door `docker exec -it n8n-n8n-1 sh` voor de n8n-instantie.

## Backup

De container is opgenomen in de wekelijkse cluster backupjob (zondag 03:00, zstd, vier weken retentie). Dit vangt het volledige container-bestandssysteem inclusief de Docker volumes met workflowdefinities en de PostgreSQL database.

Voor extra veiligheid kunnen n8n-workflowdefinities als JSON geexporteerd worden vanuit de web UI.
